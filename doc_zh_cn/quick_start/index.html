快速上手

既然说这是一个tinylib，那么就让我们用最简短的篇幅介绍一一下最基本的使用方法。

创建并保存图像

废话的少说，直接上代码：

/* test.c */
#include "tlibmp.h"
#include <stddef.h>

int main(void){
    tlb_image_t * image = NULL;

    /* Create a new image */
    image = tlb_img_new(200,100,tlb_rgb(0,0,0));

    /* Save this bmp */
    tlb_save_bmp(image,"out.bmp");

    return 0;
}

编译并运行：

cc -std=c99 -O2 -Wall -Wextra test.c tlibmp.c -o test
./test

即可得到输出图片：

/* 画外音：漆黑一片什么都没有嘛 */

首先我们调用了tlb_img_new()创建了一个大小为200*100的空白图像，图像的背景色为RGB(0,0,0)（黑色）。

tlb_img_new()函数接受三个参数，分别为图片的宽度，高度和背景色。

接下来我们调用了tlb_save_bmp()保存了这个图像为out.bmp。

tlb_save_bmp()函数接受两个参数，分别为图片数据指针和文件名字符串。

修改图片中的像素点

下一步我们要开始编辑我们的图像了，（为了简单起见，这一节我们只介绍最简单的像素修改API）

接下来的代码我们首先要重复上面的操作，然后在图片中修改一个像素点的颜色。

/* test2.c */
#include "tlibmp.h"
#include <stddef.h>

int main(void){
    tlb_image_t * image = NULL;

    /* Create a new image */
    image = tlb_img_new(200,100,tlb_rgb(0,0,0));

    /* Set the color of point (50,25) */
    tlb_pixel_set(image,50,25,tlb_rgb(255,255,255));

    /* Save this bmp */
    tlb_save_bmp(image,"out.bmp");

    return 0;
}

再次编译运行后我们得到了第out.bmp的ver 0.2

图片偏左下方的小白点，看到了么？

说你呢，别擦了，不是屏幕脏了……

我们只点了这一个像素当然看起来就像是屏幕脏了啊😂

这个例子中我们调用了tlb_pixel_set()，将点(50,25)设置为了RGB(255,255,255)（白色）。

tlb_pixel_set()函数接受三个参数，分别为图片数据指针，点的坐标和这一点要设置的颜色。

通过这个例子我们还可以知道，在这个操作库中，xy坐标原点在左下方

释放图像实例

小心内存泄漏

因为C语言没有垃圾回收机制，我们需要手工回收分配给图像的内存空间。

在上面的例子中，我们都没有进行内存释放操作，但是在实际的程序中这是万万不可取的。

带有释放内存操作的test2.c应该是这样的：

/* test3.c */
#include "tlibmp.h"
#include <stddef.h>

int main(void){
    tlb_image_t * image = NULL;

    /* Create a new image */
    image = tlb_img_new(200,100,tlb_rgb(0,0,0));

    /* Set the color of point (50,25) */
    tlb_pixel_set(image,50,25,tlb_rgb(255,255,255));

    /* Save this bmp */
    tlb_save_bmp(image,"out.bmp");

    /* Free alloced memory */
    tlb_img_free(image);

    return 0;
}

在这个例子中我们调用了tlb_img_free()来释放内存。

tlb_img_free()接受一个tlb_image_t类型指针作为唯一传入参数。

载入图像与读取像素点

既然叫libmp了，读取不能岂不是很无趣？

在这一部分我们将要载入并读取图像的某个像素点。

作为国际惯例，我们依旧使用P站 (ˉ﹃ˉ)EM1234 友情提供的这张图像：

代码如下：

/* test3.c */
#include "tlibmp.h"
#include <stdio.h>

int main(void)
{
    tlb_image_t * image = NULL;

    /* Load a bmp file */
    image = tlb_load_bmp("test.bmp");

    /* color o output */
    printf("(0,0) pixel color: #%02X%02X%02X\n",\
        tlb_pixel(image, 0, 0)[CHANNEL_R],\
        tlb_pixel(image, 0, 0)[CHANNEL_G],\
        tlb_pixel(image, 0, 0)[CHANNEL_B]);

    tlb_img_free(image);

    return 0;
}

我们得到的输出是：

(0,0) pixel color: #233A1E

不信的同学可以自己尝试打开一张图片然后和图像处理软件中取色的结果对比。

这里面我们首先调用tlb_load_bmp()载入了一张bmp图像，相信用法都能看懂，不用我再多说了。

读取像素内容的时候我们调用了tlb_pixel()函数。这个函数接受三个传入参数，无用多说，就是图片数据指针和点的坐标。返回值是一个包含了三个像素的RGB值的数组，通过[CHANNEL_R]来选择其中的红色通道。其他通道以此类推。
